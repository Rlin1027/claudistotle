# Research Pipeline: Detailed Stage Guide

## Stage 1: Research Question Formulation

### The Anatomy of a Good Research Question

A strong humanities/philosophy research question has these properties:

| Property | Bad Example | Good Example |
|----------|------------|--------------|
| **Specific** | "What is justice?" | "How does Rawls's veil of ignorance fail to account for epistemic injustice?" |
| **Debatable** | "Kant wrote the Critique of Pure Reason" | "Does Kant's transcendental deduction succeed without the thing-in-itself?" |
| **Answerable** | "Is God real?" | "How does Plantinga's reformed epistemology respond to the evidentialist challenge?" |
| **Significant** | "What color was Nietzsche's mustache?" | "How does Nietzsche's genealogical method differ from Foucault's?" |

### Question Development Process

1. **Start with curiosity**: What genuinely puzzles you about a text, concept, or debate?
2. **Find the tension**: Where do two plausible positions conflict? Where does a respected argument seem to fail?
3. **Locate in literature**: Is anyone else asking this? What's already been said?
4. **Narrow relentlessly**: From a field (ethics) → a debate (moral realism) → a specific claim (Mackie's error theory) → a specific problem (the queerness argument's reliance on empiricism)
5. **State as a question**: Use "How," "Why," "To what extent," or "In what sense" formulations

## Stage 2: Literature Review Strategy

### Search Strategy by Source Type

| Source Type | Where to Search | When to Use |
|------------|----------------|-------------|
| **Philosophy papers** | PhilPapers, JSTOR, Project MUSE | Primary scholarly sources |
| **Encyclopedia entries** | Stanford Encyclopedia of Philosophy (SEP), IEP | Orientation and key references |
| **Books & monographs** | WorldCat, Google Books, library catalogs | Deep treatments of topics |
| **Classical texts** | Perseus Digital Library, CBETA (Chinese Buddhist), CTP (Chinese Text Project) | Primary source analysis |
| **Theses** | ProQuest Dissertations, NDLTD | Recent cutting-edge work |
| **Conference proceedings** | PhilEvents, university repositories | Emerging research |

### The Synthesis Matrix

Organize literature using a thematic matrix, not a source-by-source list:

```
| Theme/Claim        | Author A | Author B | Author C | Gap?  |
|--------------------|----------|----------|----------|-------|
| Definition of X    | View 1   | View 2   | —        | No    |
| Relationship X-Y   | Agrees   | Disagrees| Nuances  | No    |
| Implication for Z  | —        | Explored | —        | YES → your contribution |
```

### Critical Reading Protocol

For each key source, extract:

1. **Central thesis** (one sentence)
2. **Key premises** (numbered list)
3. **Method used**
4. **Strongest evidence/argument**
5. **Weakest point / most vulnerable assumption**
6. **Relationship to your thesis** (supports / challenges / provides framework / irrelevant)

## Stage 3: Writing Process

### The Argument Skeleton Method

Before writing prose, build an argument skeleton:

```
THESIS: [One clear sentence]

PREMISE 1: [Claim]
  SUPPORT: [Evidence/reasoning]
  POSSIBLE OBJECTION: [Counter]
  RESPONSE: [Reply to counter]

PREMISE 2: [Claim]
  SUPPORT: [Evidence/reasoning]
  ...

CONCLUSION: [How premises entail thesis]
```

### Paragraph Architecture

Each body paragraph follows this structure:

1. **Topic sentence**: State the paragraph's claim (connects to thesis)
2. **Elaboration**: Explain what you mean
3. **Evidence/Argument**: Cite sources, provide reasoning, or analyze text
4. **Analysis**: Explain WHY the evidence supports your claim
5. **Transition**: Link to the next paragraph's claim

### Section Drafting Workflow

For each section, follow:

1. List 5-15 key points to cover
2. Arrange in logical sequence (not chronological unless analyzing historical development)
3. Draft each paragraph following the architecture above
4. Check: does every paragraph connect to the thesis?
5. Revise for clarity and concision

## Stage 4: Peer Review Simulation

### Review Criteria Weights

| Criterion | Weight | What to Check |
|-----------|--------|---------------|
| Argument validity | 30% | Logical soundness, no informal fallacies, premises support conclusion |
| Originality | 25% | Novel contribution, not mere summary of existing views |
| Scholarship | 20% | Adequate literature engagement, key sources not missed |
| Clarity | 15% | Readable prose, clear structure, well-defined terms |
| Methodology | 10% | Appropriate and consistently applied method |

### Common Reviewer Objections in Philosophy

Prepare for these common criticisms:

1. "The author has not adequately engaged with [key thinker]"
2. "The distinction between X and Y is not sufficiently clear"
3. "This argument proves too much / too little"
4. "The author conflates X with Y"
5. "An obvious counterexample undermines premise N"
6. "The charitable reading of the opponent's view is stronger than presented"
7. "The paper's contribution beyond existing literature is unclear"

## Stage 5: Revision Strategy

The change record template (COMMENT → RESPONSE → CHANGE → RATIONALE) is defined in the `peer-review` skill. Refer there for the full format.

### Revision Priority Matrix

| Severity | Type | Action |
|----------|------|--------|
| **Critical** | Logical error, missing key literature, method failure | Must fix immediately |
| **Important** | Unclear argument, weak counterargument response, structural issue | Fix in this round |
| **Minor** | Word choice, citation format, sentence-level clarity | Fix after substance is solid |
| **Stylistic** | Preferences (passive voice, hedging level) | Address only if journal requires |
